<workflow>
1. Draft `slides_plan.md` with one action-title and one audience takeaway per slide. Keep user claims and supplied data verbatim.
2. Choose HTML-native for every slide unless an explicit visual-master exception is justified.
3. Choose a style and recipe, then query compatible layouts. Inspect only selected contracts and validate copy and media capacity before generation.
4. Build a fixed 1920×1080 deck using `.slide`, `.active`/`.visible`, `data-slide-id`, and `data-object-id`.
5. Add DiagramSpec or MotionProgram as versioned JSON when needed; do not hard-wire anonymous animations.
6. Build author and share HTML. Open the share build at desktop and phone viewport; phone may letterbox but must not reflow.
7. Run validation and export only the formats requested.
