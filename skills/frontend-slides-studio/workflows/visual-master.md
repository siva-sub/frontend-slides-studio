<workflow name="visual-master">
<required_reading>
- Read `references/product-map.md`, `workflows/assets.md`, and `references/contracts.md`.
</required_reading>

<process>
1. Confirm the slide is art-directed and not text, data, legal, table, or evidence dense.
2. Record supplied assets and mark fidelity-critical media for preserved independent overlay, never model redrawing by default.
3. Select an original style/layout contract and calculate the final real-asset box from aspect ratio and detail level.
4. Generate a transparent corner-marker skeleton for the exact box, then generate only the visual background.
5. Overlay the original asset as an independent HTML/PPTX object and store prompt, references, geometry, provider metadata, and trace preview.
6. For editable reconstruction, retain visual master, clean plate, masks, scene manifest, A1/A2/B layer-route provenance, and zero-change-outside-mask evidence.
7. Load the resulting slide in Studio, inspect placement and readability, save, reload, and run rendered quality.
8. Keep status at `rendered_pending_manual_review` until a named reviewer views a fresh rendered result.
</process>

<success_criteria>
- Visual-master use is explicit and justified.
- Fidelity-critical assets remain independent overlays.
- Studio review and evidence bundle remain linked to the saved revision.
</success_criteria>
</workflow>
