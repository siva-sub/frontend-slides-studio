# Visual reconstruction example

A deterministic, self-contained compact reconstruction bundle produced entirely
by this repository. It demonstrates the contained visual-master pipeline:

- a tiny synthetic **visual master** (64×36),
- a **clean plate**,
- a **repair mask** + per-subject **layer** with an A1/A2/B route decision,
- **edge-check** composites (white/black backgrounds),
- an **overlay trace** from declared candidate placement,
- a **scene** manifest and a **quality report** with an honest review state.

## Regenerating

From the repository root:

```bash
python3 examples/visual-reconstruction/generate.py
```

The generator is deterministic: two runs produce byte-identical images and
identical JSON. Pass a target directory to write the bundle elsewhere:

```bash
python3 examples/visual-reconstruction/generate.py /tmp/example-bundle
```

## Files

| File | Description |
| --- | --- |
| `visual-master.png` | Synthetic source slide. |
| `clean-plate.png` | Slide with the repair region composited. |
| `repair-mask.png` | White = replace, black = preserve. |
| `layers/subject.png` | Extracted subject layer (route recorded in the manifest). |
| `edge-check-white.png` / `edge-check-black.png` | Subject over white/black backgrounds. |
| `overlay-trace.png` | Selected overlay region + corner skeleton trace. |
| `scene.json` | Contained scene manifest. |
| `quality-report.json` | Actionable metrics and honest review state. |
| `evidence-manifest.json` | Full evidence bundle (hashes, routes, dimensions). |

The review status is `rendered_pending_manual_review` (evidence present, manual
review pending) — it is never auto-set to `passed`.
